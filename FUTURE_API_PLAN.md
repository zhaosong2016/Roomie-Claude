## 起点：王煜全 3月28日演讲

王煜全在《生财有术》大会的演讲指出了一个方向：

**未来基于兴趣的社交，会比基于关系的社交更重要。**

人们会因为共同参与某件事（音乐节、体育赛事、学习活动）聚在一起，形成真实的连接。这不是微信那种"我认识你"的关系链，而是"我们做过同一件事"的场景链。

这场演讲让我们重新看待 Space One 拼房实验室——它不只是一个拼房工具，它是一个**场景入口**。

---

## 两层结构

### 第一层：拼房 API 作为基座

现有的 `/api/*` 接口已经是 API-First 的结构。网页只是其中一个调用方，任何人（其他 App、AI 助手、活动平台）都可以直接调用同一套接口。

这意味着拼房能力可以变成**基础设施**：
- 任何活动组织者都可以接入，生成口令，让参与者自助匹配
- 不依赖某一个具体页面或品牌
- 数据在我们这里积累

### 第二层：基于兴趣的社交平台

拼房是切入点，不是终点。

当足够多的人通过 Space One 在同一个场景下找到室友，就开始有了社区的雏形：
- 去过同一个音乐节的人
- 参加过同一个考察团的人
- 在同一个峰会拼过房的人

这些人有共同的经历、共同的话题、共同的兴趣。这是比"我们是校友"或"我们是同事"更活跃的关系基础。

长远来看，Space One 可以从一个匹配工具，生长成一个**基于真实共同经历的兴趣社区**。

## 为什么音乐节是第一个国际场景

- Coachella、Tomorrowland、EDC Las Vegas——每次几万人，找室友需求非常真实
- 目前没有专属工具，大家靠 Reddit、Facebook 群、小红书刷屏
- 场景标准化：入住日期固定、人群相似、愿意为好的体验付费
- 验证付费意愿比国内会议容易

草莓音乐节（武汉，4月5日）已经在小红书投放了第一条帖子，口令 CMWH0405，作为最早的市场验证。

## AI 圆桌·终局技术架构（2026-04-05）

圆桌给出了一个三层架构蓝图，整体方向是对的，但有一个判断需要修正。

### 三层架构

**顶层（API Gateway）**：面向未来的个人 AI 助手（Siri、ChatGPT 等），接收自然语言意图，比如"预算300，找5月1日草莓音乐节拼房"。

**中层（信任与匹配引擎）**：这是核心 IP。
- 兴趣/行为模型：接收用户特征向量，在向量空间里计算匹配度
- 信任与安全层：历史好评、芝麻信用、身份验证
- 分账系统：联合钱包，双方各打入50%

**底层（物理执行基座）**：酒店/房源 API、资金清算、押金管理，将这次入住的行为评价反哺给信任引擎。

### 关于"必须掌控酒店 API 和清算"的修正

圆桌说"必须掌控底层，否则被白嫖"——逻辑对，但**顺序错了**。

酒店 API 对接、资金托管、清算合规，每一件都是重资产+强监管，和"不碰重资产"的原则直接矛盾。现阶段去做底层，是把精力用在了错的地方。

**正确的顺序：**
1. 先把中层做深——兴趣向量 + 信任引擎，这是唯一的核心 IP
2. 中层有价值了，顶层（AI 助手接入）自然来找你
3. 底层（资金 + 酒店 API）是最后封装的，而且到时候是合作而非自建

接酒店是必须要做的，但不是现在的主要工作。

## 当前阶段：市场已验证，下一步挖数据（2026-04-05）

### 市场验证结论

杭州"生财有术"1000人群体，其中100人主动来用，说明需求是真实的。1000人里约一半不需要拼房，另一半里有100人响应——转化率健康，不是小众需求。

**现在要做的不是再验证需求，而是加深数据维度。**

### 下一步：扩展特征向量

现有维度（生存层）：性别、日期、吸烟、作息——解决"不出事"

下一步要收集的（化学反应层）：
- **睡眠数据**：不只是早睡晚睡，还有敏感度、规律性
- **性格数据**：内向/外向、整洁度、对噪音的容忍度
- **兴趣向量**：音乐口味、活动类型偏好，未来可以接入歌单（Spotify/Apple Music）做行为验证

关键原则：**行为证明的数据比自填问卷可信**。用户说"我喜欢摇滚"没有信息量，他歌单里循环了多少次才是真实信号。问卷是起点，行为数据是终点。

### Cohabitation Credit 积累从现在开始

每一次拼房成功后，要开始记录：互评、容忍度反馈、是否成为长期朋友。这些数据只有在我们这里才会产生，是三五年后真正的护城河。

## AI 圆桌结论（2026-04-05）

用 AI 做了一次圆桌讨论，核心结论记录如下。

### 天花板判断

- **低配（千万级）**：一个好用的音乐节拼房工具。受限于低频、安全摩擦力，小而美。
- **高配（百亿美金）**：Z 世代线下共享信任协议（Trust Protocol）。当所有 AI 助手需要安排两个陌生人共享空间时，调用我们的 API 获取"信任许可和匹配建议"。

### 明确不做的事

**掌握独立供给**（包下公寓、改造成粉丝基地）——重资产、强运营，和 API 平台方向相反，现阶段不碰。

### 核心定位调整

不和携程/美团抢订房利润。定位是**"同好 + 安全"**，而非"帮你省房费"。
- 省钱是让人来的理由（获客）
- 同好是让人留下来、愿意再来的理由（口碑）

### 匹配维度的演化路径

**现在（生存层）**：性别 + 日期 + 吸烟 + 作息 → 不出事

**中期（化学反应层）**：加入兴趣维度 → 出好事

**长期（行为数据层）**：用行为替代问卷，计算多维特征向量的余弦相似度

关键洞察：**行为证明的兴趣，比自己填写的兴趣可信得多。**
- "我喜欢 Taylor Swift"——没有信息量，人人都这么说
- 歌单里循环了多少次哪首歌——这是真实的人

两个都爱 Taylor Swift 的人，一个循环 Folklore，一个循环 Shake It Off，性格可能完全不同。歌单不只说明喜好，还说明性格。

未来如果做"授权接入歌单"（Spotify / Apple Music），一个人的兴趣向量立刻从5个维度变成500个——而且全是行为证明的，不是自己填的。

### Cohabitation Credit（同住信用）

芝麻信用衡量会不会还钱，但不能告诉你这个人睡觉打不打呼、会不会把浴室弄乱。

**从今天起就要记录**：拼房后的互评、容忍度、是否成为长期朋友。这些数据只有在我们这个场景里才会产生，是未来真正的护城河。巨头不会把住客的歌单和同住行为关联起来——只有我们会。
