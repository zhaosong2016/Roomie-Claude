# Space One · 未来方向

> 记录于 2026-04-02，4月5日补充 AI 圆桌讨论结论

---

## 起点：王煜全 3月28日演讲

王煜全在《生财有术》大会的演讲指出了一个方向：

**未来基于兴趣的社交，会比基于关系的社交更重要。**

人们会因为共同参与某件事（音乐节、体育赛事、学习活动）聚在一起，形成真实的连接。这不是微信那种"我认识你"的关系链，而是"我们做过同一件事"的场景链。

这场演讲让我们重新看待 Space One 拼房实验室——它不只是一个拼房工具，它是一个**场景入口**。

---

## 两层结构

### 第一层：拼房 API 作为基座

现有的 `/api/*` 接口已经是 API-First 的结构。网页只是其中一个调用方，任何人（其他 App、AI 助手、活动平台）都可以直接调用同一套接口。

这意味着拼房能力可以变成**基础设施**：
- 任何活动组织者都可以接入，生成口令，让参与者自助匹配
- 不依赖某一个具体页面或品牌
- 数据在我们这里积累

### 第二层：基于兴趣的社交平台

拼房是切入点，不是终点。

当足够多的人通过 Space One 在同一个场景下找到室友，就开始有了社区的雏形：
- 去过同一个音乐节的人
- 参加过同一个考察团的人
- 在同一个峰会拼过房的人

这些人有共同的经历、共同的话题、共同的兴趣。这是比"我们是校友"或"我们是同事"更活跃的关系基础。

长远来看，Space One 可以从一个匹配工具，生长成一个**基于真实共同经历的兴趣社区**。

## 为什么音乐节是第一个国际场景

- Coachella、Tomorrowland、EDC Las Vegas——每次几万人，找室友需求非常真实
- 目前没有专属工具，大家靠 Reddit、Facebook 群、小红书刷屏
- 场景标准化：入住日期固定、人群相似、愿意为好的体验付费
- 验证付费意愿比国内会议容易

草莓音乐节（武汉，4月5日）已经在小红书投放了第一条帖子，口令 CMWH0405，作为最早的市场验证。

## AI 圆桌结论（2026-04-05）

用 AI 做了一次圆桌讨论，核心结论记录如下。

### 天花板判断

- **低配（千万级）**：一个好用的音乐节拼房工具。受限于低频、安全摩擦力，小而美。
- **高配（百亿美金）**：Z 世代线下共享信任协议（Trust Protocol）。当所有 AI 助手需要安排两个陌生人共享空间时，调用我们的 API 获取"信任许可和匹配建议"。

### 明确不做的事

**掌握独立供给**（包下公寓、改造成粉丝基地）——重资产、强运营，和 API 平台方向相反，现阶段不碰。

### 核心定位调整

不和携程/美团抢订房利润。定位是**"同好 + 安全"**，而非"帮你省房费"。
- 省钱是让人来的理由（获客）
- 同好是让人留下来、愿意再来的理由（口碑）

### 匹配维度的演化路径

**现在（生存层）**：性别 + 日期 + 吸烟 + 作息 → 不出事

**中期（化学反应层）**：加入兴趣维度 → 出好事

**长期（行为数据层）**：用行为替代问卷，计算多维特征向量的余弦相似度

关键洞察：**行为证明的兴趣，比自己填写的兴趣可信得多。**
- "我喜欢 Taylor Swift"——没有信息量，人人都这么说
- 歌单里循环了多少次哪首歌——这是真实的人

两个都爱 Taylor Swift 的人，一个循环 Folklore，一个循环 Shake It Off，性格可能完全不同。歌单不只说明喜好，还说明性格。

未来如果做"授权接入歌单"（Spotify / Apple Music），一个人的兴趣向量立刻从5个维度变成500个——而且全是行为证明的，不是自己填的。

### Cohabitation Credit（同住信用）

芝麻信用衡量会不会还钱，但不能告诉你这个人睡觉打不打呼、会不会把浴室弄乱。

**从今天起就要记录**：拼房后的互评、容忍度、是否成为长期朋友。这些数据只有在我们这个场景里才会产生，是未来真正的护城河。巨头不会把住客的歌单和同住行为关联起来——只有我们会。
