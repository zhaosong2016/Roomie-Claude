# Space One · 未来方向

> 记录于 2026-04-02

---

## 起点：王煜全 3月28日演讲

王煜全在《生财有术》大会的演讲指出了一个方向：

**未来基于兴趣的社交，会比基于关系的社交更重要。**

人们会因为共同参与某件事（音乐节、体育赛事、学习活动）聚在一起，形成真实的连接。这不是微信那种"我认识你"的关系链，而是"我们做过同一件事"的场景链。

这场演讲让我们重新看待 Space One 拼房实验室——它不只是一个拼房工具，它是一个**场景入口**。

---

## 两层结构

### 第一层：拼房 API 作为基座

现有的 `/api/*` 接口已经是 API-First 的结构。网页只是其中一个调用方，任何人（其他 App、AI 助手、活动平台）都可以直接调用同一套接口。

这意味着拼房能力可以变成**基础设施**：
- 任何活动组织者都可以接入，生成口令，让参与者自助匹配
- 不依赖某一个具体页面或品牌
- 数据在我们这里积累

### 第二层：基于兴趣的社交平台

拼房是切入点，不是终点。

当足够多的人通过 Space One 在同一个场景下找到室友，就开始有了社区的雏形：
- 去过同一个音乐节的人
- 参加过同一个考察团的人
- 在同一个峰会拼过房的人

这些人有共同的经历、共同的话题、共同的兴趣。这是比"我们是校友"或"我们是同事"更活跃的关系基础。

长远来看，Space One 可以从一个匹配工具，生长成一个**基于真实共同经历的兴趣社区**。

## 为什么音乐节是第一个国际场景

- Coachella、Tomorrowland、EDC Las Vegas——每次几万人，找室友需求非常真实
- 目前没有专属工具，大家靠 Reddit、Facebook 群、小红书刷屏
- 场景标准化：入住日期固定、人群相似、愿意为好的体验付费
- 验证付费意愿比国内会议容易

草莓音乐节（武汉，4月5日）已经在小红书投放了第一条帖子，口令 CMWH0405，作为最早的市场验证。

## 分阶段计划

### 现在（2026上半年）
- 用国内活动场景继续积累数据和反馈
- 把现有系统做扎实（协议、反馈、匹配稳定性）
- 5月硅谷考察团做完，6月后全职投入

### 国际化准备（2026下半年）
- 英文界面
- 支付能力（Paddle，绕过大陆主体限制）
- 身份验证（Stripe Identity 或类似方案）
- 嵌入式保险（每笔交易附加）

### 第一个国际场景（时间待定）
- 选定一个音乐节，冷启动
- 验证付费和口碑传播
